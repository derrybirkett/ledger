How it works

1

Five deterministic rules run every morning

Each rule checks one pipeline handoff — Delta→PR, PR→Council, Council→Merge, Merge→merged. Rules are stateless: all state lives in GitHub labels and live API calls. No database, no config needed.

2

Claude writes a plain-English diagnosis

When gaps are found, the raw rule output is handed to Claude, which writes a concise morning briefing — one diagnosis and one concrete action per gap. No hedging, no jargon.

3

A ledger/gap issue lands in your inbox

The report is opened as a GitHub issue with the ledger/gap label, including the full diagnosis and a live pipeline state table. If no gaps: silent exit, no issue.

Gap types monitored

Rule What it checks
Delta ran when it should skip Delta opened a PR despite existing open delta PRs — duplicate work
Delta failed when it should run Delta failed with no open PRs and a non-empty backlog — blocked cycle
Merge window expired PR had merge/ready for >48h but wasn't merged — Merge agent down
Council approved, merge/ready missing council/approved without merge/ready — integration hook not installed
Council review overdue Delta PR open >24h with no Council review — Council not running

Install

bash 
# 1. Add Ledger as a submodule
git submodule add https://github.com/derrybirkett/ledger ledger

# 2. Copy the workflow into your repo
cp ledger/.github/workflows/ledger.yml .github/workflows/ledger.yml

# 3. Create the ledger/gap label (one-time)
bash ledger/scripts/setup.sh

# 4. Add your Anthropic API key to GitHub repo secrets
#    Settings → Secrets → ANTHROPIC_API_KEY

# 5. Commit and push — audits fire at 8am UTC daily
git add .gitmodules ledger .github/workflows/ledger.yml
git commit -m "chore: install ledger"
git push

Requires ANTHROPIC_API_KEY and GITHUB_TOKEN (automatically available in Actions). Trigger a test run with gh workflow run ledger.yml.
